# 🌊 Kabini River Daily Discharge Forecasting

End-to-end ML pipeline for predicting daily river discharge at the Kabini upstream gauging station (Karnataka, India), combining classical hydrological analysis with machine learning and CMIP6 climate projections.

## Quick Start

### 1. Install dependencies

```bash
python -m venv venv
source venv/bin/activate  # macOS/Linux
pip install pandas numpy matplotlib scikit-learn xgboost lightgbm optuna tensorflow
pip install streamlit pillow
```

### 2. Prepare data

Place `master_dataset.csv` and `chirps_kabini_daily.csv` in the `data/` directory. Required columns:

| File | Required Columns |
|------|-----------------|
| `master_dataset.csv` | `date`, `q_upstream_mk`, `rainfall_max_mm` |
| `chirps_kabini_daily.csv` | `date`, `rainfall_std_mm` |

### 3. Run the pipeline

```bash
# Option A: Run scripts individually
python eda.py
python hydrology_analysis.py
python feature_engineering.py
python train_model.py
python forecast.py

# Option B: Use the Streamlit dashboard
streamlit run app.py
```

### 4. CMIP6 data (for forecasting)

The forecast script requires CMIP6 precipitation data. Generate it using Google Colab:

1. Upload `cmip6_colab.py` to [Google Colab](https://colab.research.google.com/)
2. Upload your `master_dataset.csv` when prompted
3. The script downloads NEX-GDDP-CMIP6 data from AWS S3, bias-corrects it, and saves CSVs
4. Download `forecast_input_ssp245.csv` and `forecast_input_ssp585.csv`
5. Place them in `cmip6_data/processed/`
6. Run `python forecast.py`

## Pipeline Phases

### Phase 1: EDA (`eda.py`)
- Rainfall and discharge distributions
- Time-series visualisation
- Correlation matrix
- Seasonal decomposition

### Phase 2: Hydrology Analysis (`hydrology_analysis.py`)
- **Baseflow separation** — Eckhardt (2005) recursive digital filter → BFI
- **Recession analysis** — Exponential decay fitting → recession constant, half-life
- **Unit hydrograph** — Event-based derivation → Tp, Qp, Tb + SCS UH comparison
- **SCS Curve Number** — Back-calculated from observed P-Q pairs (Ia = 0.3S for India)
- **Water balance** — Annual runoff coefficients
- **Time of concentration** — Kirpich formula + observed lag
- **Flow duration curve** — Q50, Q75, Q90, Q95 percentiles
- **Flood frequency** — Gumbel distribution, 2–100 year return periods

### Phase 3: Feature Engineering (`feature_engineering.py`)
34 features in 12 families:

| Family | Count | Description |
|--------|-------|-------------|
| Rainfall lags (log) | 7 | 1–7 day shifted rainfall |
| Rolling rainfall sums (log) | 4 | 3/7/14/30-day windows |
| Rolling rainfall std (log) | 2 | 7/14-day variability |
| Discharge lags (log) | 3 | 1–3 day auto-regression |
| Discharge rate of change | 2 | Rising/falling momentum |
| EWM rainfall (log) | 3 | Exponentially weighted (spans 3/7/14) |
| Rain × discharge interaction (log) | 2 | Nonlinear runoff amplification |
| Rolling discharge stats (log) | 4 | 7/14-day mean and std |
| Consecutive dry days (log) | 1 | Soil moisture proxy |
| Seasonality (sin/cos) | 4 | Month + day-of-year encoding |
| Monsoon flag | 1 | June–September binary |
| CHIRPS spatial std (log) | 1 | Rainfall heterogeneity |

Target: `log_q = log1p(q_upstream_mk)` — reversed with `expm1` after prediction.

### Phase 4: Model Training (`train_model.py`)

**Temporal split** (no shuffling — strict chronological order):
- **Train:** Start → 2011-12-31
- **Validation:** 2012-01-01 → 2014-12-31
- **Test:** 2015-01-01 → End

**Models compared:**

| Model | RMSE (m³/s) | MAE (m³/s) | NSE |
|-------|-------------|------------|-----|
| **Ridge** | **63.01** | **18.96** | **0.8897** |
| XGB Ensemble | 86.65 | 20.84 | 0.7914 |
| LightGBM | 88.93 | 21.03 | 0.7804 |
| XGBoost | 91.69 | 21.41 | 0.7665 |
| Hybrid (XGB+LSTM) | 93.85 | 23.77 | 0.7554 |
| LSTM | 104.61 | 29.20 | 0.6961 |

Ridge wins because the dominant signal is the near-linear autoregressive relationship between consecutive days' discharge in log-space.

**Methodological safeguards:**
- Scaler fitted on training data only (no test leakage)
- Ensemble weights from train-only models (no circular prediction)
- Test set never visible during training (no eval_set leakage)
- Unidirectional LSTM (no future context from bidirectional pass)

### Phase 5: Forecasting (`forecast.py`)

Auto-regressive day-by-day prediction to 2030 using:
- **SSP2-4.5** — "Middle of the road" moderate emissions
- **SSP5-8.5** — "Fossil-fueled development" high emissions

Rainfall from NEX-GDDP-CMIP6 (3-model ensemble: ACCESS-CM2, MIROC6, MPI-ESM1-2-HR), bias-corrected against CHIRPS via monthly quantile mapping.

## Kabini Basin

- **Location:** 11.60–12.25°N, 75.80–76.50°E
- **Catchment area:** ~2,100 km²
- **Origin:** Wayanad/Brahmagiri hills, Western Ghats (Kerala)
- **Dam:** Kabini Reservoir (Mysore District, Karnataka)
- **Confluence:** Joins Cauvery near T. Narsipur
- **Climate:** SW monsoon dominant (June–September), 1000–1500 mm annual rainfall
